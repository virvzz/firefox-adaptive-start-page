Техническое задание (ТЗ)
Проект

Название (рабочее): Firefox Adaptive Start Page (FASP)

Тип проекта: Browser Extension for Mozilla Firefox

Цель проекта:

Разработать расширение для Firefox, полностью заменяющее стандартную стартовую страницу браузера (new tab page), предоставляющее пользователю высоко настраиваемый визуальный интерфейс с быстрым доступом к сайтам, папкам закладок, динамическим фоном, адаптивной сеткой элементов и синхронизацией настроек между устройствами.

1. Функциональные требования
1.1 Главный экран (New Tab Page)

При открытии новой вкладки пользователь должен видеть кастомную стартовую страницу.

Содержимое:

сетка плиток быстрого доступа;
фоновое изображение/генерируемый фон;
строка поиска (опционально);
часы (опционально);
виджеты (опционально).
1.2 Система плиток (Tiles System)

Плитка представляет собой:

отдельный сайт;
папку закладок;
группу сайтов;
внутреннюю папку с вложенными плитками.

Каждая плитка содержит:

favicon сайта;
screenshot-preview страницы;
пользовательское изображение (optional);
название.

При клике:

Для обычного сайта:

Открывается URL.

Для папки:

Открывается popup/grid overlay со вложенными плитками.

Пример:

[YouTube] [GitHub] [Work ▼]

Нажатие Work →

 ----------------------
 | Jira    | Jenkins |
 | Grafana | Docs    |
 ----------------------
1.3 Интеграция с Firefox Bookmarks

Расширение должно читать:

Firefox Sync bookmark tree.

Пользователь может:

импортировать закладки;
импортировать целые папки;
отображать папку как одну плитку;
автоматически обновлять содержимое.

Важно:

изменения в bookmark tree должны подтягиваться автоматически.

1.4 Layout Engine

Пользователь должен настраивать:

количество плиток в ряду (2–12);
spacing между плитками;
adaptive scaling;
размеры плитки.

Пример:

2 columns → large tiles

8 columns → small tiles

Логика:

tile_width = screen_width / columns - spacing

Настройки меняются:

через UI slider;
без ручного редактирования config.
2. Preview Engine (Автоэскизы сайтов)

При добавлении сайта:

Система автоматически:

открывает страницу headless;
делает screenshot;
сохраняет thumbnail.

Форматы:

256x144
WebP preferred

Fallback:

Если screenshot не удалось получить:

favicon
default placeholder

Пользователь может:

заменить вручную.
3. Background Engine

Два режима.

Mode A — Internet Wallpapers

Автоматическая загрузка обоев.

Источники:

Unsplash API
Wallhaven API
Pexels API

Настройки:

категория;
blur;
brightness;
update interval.
Mode B — Mathematical Generative Art

Фон генерируется алгоритмически.

Варианты:

Fractal Noise

Perlin Noise.

Particle Simulation

Роевые частицы.

Julia Set

Фракталы.

Cellular Automata

Наподобие Conway.

Reaction Diffusion

Паттерны как у живых систем.

Параметры:

FPS limiter;
GPU acceleration;
animation on/off.
Mode C — Static Background

Пользователь загружает:

jpg
png
webp
4. Tile Appearance Engine

Автоматическое создание карточек.

После добавления URL:

Fetch favicon
↓
Generate screenshot
↓
Extract dominant colors
↓
Create card theme

Можно задать:

собственное изображение;
прозрачность;
border radius;
glassmorphism.
5. Synchronization

Синхронизация между устройствами.

Через:

Firefox Sync storage.

Должны синхронизироваться:

layout;
tiles;
folders;
background settings;
custom images metadata.
Альтернатива

Cloud sync providers.

Поддержка:

Dropbox
Google Drive
Nextcloud
6. Performance Requirements

Максимальное время открытия вкладки:

< 300 ms

Потребление RAM:

< 250 MB

CPU usage idle:

< 2%

Background animation:

60 FPS target
30 FPS fallback

Lazy loading:

обязательно.

7. UX Requirements

Drag & Drop:

перемещение плиток;
перемещение папок;
reorder.

Right click menu:

edit;
remove;
regenerate preview;
change icon.

Hover animation.

Smooth transitions.

8. Privacy Requirements

Расширение не должно отправлять:

browsing history;
URLs;
cookies;
telemetry.

Внешние API:

только по explicit consent.

Offline mode обязателен.
