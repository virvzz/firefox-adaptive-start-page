# Adaptive Start Page - сводка продукта

Adaptive Start Page - персональная новая вкладка для Firefox. Она превращает стартовую страницу в настраиваемое пространство с плитками, папками, темами, обоями, экспортом/импортом профиля и интеграцией с закладками.

## Что это за продукт

Это не корпоративный дашборд и не "комбайн всего на свете". Это личная стартовая поверхность: быстрый доступ к сайтам, папкам и закладкам, с сильным акцентом на внешний вид, контроль и спокойный интерфейс.

Короткое обещание продукта:

> Красивая local-first стартовая страница Firefox, которая ощущается личной, быстрой и полностью управляемой пользователем.

## Что уже хорошо

- Плитки и папки работают и выглядят цельно.
- Drag and Drop работает, включая сценарии с папками.
- Папки из закладок поддерживают Reference и Clone.
- Темы, стеклянные поверхности, обои и генеративные фоны уже в хорошем состоянии.
- Настройки собраны в современную glass-модалку.
- Есть live preview для "Темы" и "Плитки и макет".
- Экспорт/импорт профиля сохраняет настройки, плитки, папки, порядок, темы, обои и media assets.
- Firefox Containers поддерживаются на уровне конкретной плитки.
- Можно выбрать, как открывать плитки: в текущей вкладке, новой вкладке или новом окне.
- Есть массовое перекрашивание плиток.
- Новая плитка может наследовать преобладающий цвет уже существующих плиток.
- Расширение уже подписано через AMO и используется владельцем.

## Приватность

Честная позиция:

- Нет аналитики.
- Нет сервера разработчика.
- Нет телеметрии разработчику.
- Данные хранятся локально в Firefox storage/IndexedDB.
- Экспорт/импорт профиля запускается только пользователем.
- Онлайн-эскизы и иконки опциональны и выключены по умолчанию.
- Если онлайн-эскизы/иконки включены, сторонние сервисы могут получить домены или URL плиток.

Хорошая публичная формулировка:

> Adaptive Start Page не собирает аналитику и не отправляет данные разработчику. По умолчанию превью плиток локальные. Опциональные онлайн-эскизы и иконки сайтов могут обращаться к сторонним сервисам, используя домен или URL плитки.

Чего не писать:

> Ничего никогда никуда не отправляется.

Это верно только когда выключены онлайн-превью, погода и любые другие удаленные функции.

## Возможности для пользователя

### Плитки

- Создание, редактирование, удаление.
- Своя картинка или автоматическое превью.
- Режимы: иконка сайта, эскизы, смешанный.
- Полные или компактные подписи.
- Tooltip для длинных названий.
- Цвет плитки, HEX-код, массовая перекраска.
- Firefox Container для конкретной плитки.
- Открытие в текущей вкладке, новой вкладке или новом окне.

### Папки

- Создание папок.
- Сетка или список внутри открытой папки.
- Настраиваемое число плиток в ряду внутри папок.
- Счетчик элементов.
- Бейдж REF/CLONE для папок из закладок.
- Drag and Drop внутрь и наружу.

### Темы и фон

- Встроенные и пользовательские темы.
- Стеклянный эффект, скругление, прозрачность, скорость анимации.
- Статические обои и генеративные фоны.
- Живые демонстрации в настройках.

### Синхронизация и перенос

- Локальный экспорт/импорт профиля.
- Есть транспорт через Firefox Sync.
- Профиль включает настройки, макет, плитки, папки, порядок, темы, фон и media assets.

### Дополнительно

- Оптимизация медиа.
- Адаптивная читаемость переключателей.
- Debug/export инструменты для анализа старта.

## Дистрибуция

Расширение подписывается через AMO для self-distribution. GitHub Pages используется для `updates.json`.

Текущая версия в репозитории: `0.1.7`.

Для каждого релиза:

- поднять версию;
- собрать extension archive;
- собрать source archive;
- загрузить оба архива в AMO;
- обновить GitHub/update metadata при необходимости.

## Куда развивать дальше

Лучшее направление:

- полировка, а не перегруз;
- онлайн-сервисы только опционально;
- честные и понятные формулировки;
- улучшение fallback-иконок;
- accessibility и клавиатурный фокус;
- не превращать стартовую страницу в тяжелый productivity suite.

Хорошие кандидаты на будущее:

- Разделить онлайн-превью на две настройки:
  - онлайн-иконки сайтов;
  - онлайн-эскизы страниц.
- Улучшить локальные иконки:
  - буква домена;
  - детерминированный цвет;
  - ручная загрузка иконки.
- Обновить privacy copy в AMO и README.
- Добавить smoke-проверки для fallback-иконок и privacy-поведения.

Не срочно:

- Еще больше виджетов.
- Landing-page стиль.
- Тяжелая облачная синхронизация.
- Work/life-профили.

## Личная сводка

Это уже не прототип. Это рабочее персональное расширение с цельной визуальной идентичностью и большим количеством полезной полировки. Его сильная сторона - "local-first, красиво, под контролем пользователя". Главное дальше - не растерять доверие: честная приватность, предсказуемое поведение и отсутствие лишнего перегруза.
